Reflection


Usability and Accessibility


From a usability and accessibility standpoint, the dropdown menu is designed to be easily located and clicked with a mouse. The options are intuitively arranged, allowing users to seamlessly hover and select their desired choice. To preserve this functionality, I maintained the existing design while incorporating an additional feature that enables users to return to the dropdown menu after selecting an option, facilitating multiple selections when needed.

Accessibility Considerations


A critical accessibility concern in the original design was the absence of a keyboard shortcut for accessing the workspaces menu. This limitation poses a challenge for users who rely on keyboard navigation instead of a mouse or trackpad. To address this, I introduced a keyboard shortcut, ensuring a more inclusive experience.

Solution for "Mismatch" in Our World


By implementing this enhancement, my design mitigates the "mismatch" between users’ needs and system limitations, allowing individuals who cannot use a mouse or trackpad to fully engage with the dropdown menu. Moreover, the addition of customizable shortcuts extends accessibility to voice command features, providing features for users who interact with their devices through alternative input methods to utilize the functionality seamlessly.

Examples of accessibility/inaccessibility


One notable issue is that all dropdown menu options change color to indicate selection. This design choice poses a significant challenge for users with visual impairments, such as color blindness, as they may struggle to distinguish the selected option from the rest.


Additionally, for keyboard users, exiting the dropdown menu without making a selection is unnecessarily difficult. The menu remains open until an option is chosen or the user manually clicks outside of it, creating an inconvenient experience for those relying solely on keyboard navigation.

Priority Audience of the Component


Across different input methods—mouse, keyboard, touch/mobile, and screen readers—it is evident that the design of these applications prioritizes mouse users. This imbalance negatively impacts the overall user experience, as individuals using alternative input methods face unnecessary obstacles when accessing key features.
